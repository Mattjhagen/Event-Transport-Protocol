# Contributing to ETP

We welcome contributions to the ETP protocol and its reference implementations.

## How to Contribute
1. **Spec Improvements**: Propose changes to the `/spec` folder via Pull Requests. All changes must be technically justified and maintain backward compatibility or provide a clear migration path.
2. **Bug Reports**: Use Issue trackers to report vulnerabilities or implementation flaws.
3. **Reference Code**: Improve the Hono/React reference implementation in `/src`.

## Guiding Principles
- **Minimalism**: The protocol should be as simple as possible.
- **Extensibility**: Favor metadata fields over adding rigid top-level properties.
- **Interoperability**: Always consider how a change affects legacy calendar systems (Outlook, Apple, Google).

## Code of Conduct
Focus on technical excellence and professional technical discourse.
